**Input:**

user-input: {user_input}

conversation-history: {conversation_history}


**Output (strict):**

1. Return **one JSON object only** — no markdown, no backticks, no `//` comments, no text before or after the JSON.
2. The object must be: `{{"tasks": [ task1, task2, ... ]}}`  
   If there is a single user request, `tasks` has **one** element.
3. Each task is an object with exactly these keys (valid JSON only):

| key | type | rules |
|-----|------|--------|
| `question` | string | The user’s request, rephrased if needed for clarity. |
| `function_name` | string | One of: `snowflake_agent`, `generic_conversation_agent`, `human_agent`. |
| `function_params` | **array of strings** | For `snowflake_agent` and `generic_conversation_agent` use **one** string: the full user question to send to the tool. For `human_agent` you may use multiple strings if the prompt requires it. **Never** use an object/dictionary for `function_params`. |
| `depends_on` | null or number | 1-based index of a **previous** task in this `tasks` array that this task depends on, or `null` if none. |

**Example (format only):**

`{{"tasks":[{{"question":"Show revenue by region.","function_name":"snowflake_agent","function_params":["Show revenue by region."],"depends_on":null}}]}}`

**Function selection (reminder):** Use `snowflake_agent` for data/list/table/SQL/REGION/LINEITEM-style questions. Use `generic_conversation_agent` for general chat. Use `human_agent` only if the request is too ambiguous to route otherwise.

**Notes:**  
- Rephrase the user in `function_params[0]` when the conversation context makes the phrasing clearer.  
- Do **not** put the assistant’s prior answers or large table data inside `function_params` — only the **question to run** for this turn.
